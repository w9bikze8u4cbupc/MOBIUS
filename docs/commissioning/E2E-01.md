# E2E-01: MOBIUS v1 Commissioning Run

**Purpose**: End-to-end commissioning run to validate MOBIUS v1 production readiness  
**Version**: 1.0  
**Last Updated**: 2026-02-10

## Overview

E2E-01 is a deterministic commissioning runner that executes the complete MOBIUS pipeline from ingestion through render, enforcing all governance invariants and producing a sealed commissioning report.

## Prerequisites

### System Requirements
- Node.js 18+
- MOBIUS API server running on `localhost:5001` (or custom `API_BASE_URL`)
- SQLite database initialized
- All dependencies installed (`npm ci`)

### Required Inputs
- **Project ID**: Existing project in database
- **PDF Path**: Path to game rulebook PDF (for non-dry-run)
- **BGG URL**: BoardGameGeek URL (optional, if metadata already present)

### Optional: HEPHAESTUS
- `MOBIUS_ENABLE_HEPHAESTUS=true` in `.env`
- `HEPHAESTUS_WORKSPACE` configured
- HEPHAESTUS external workspace set up

## Usage

### Dry Run (CI/Local Wiring Check)

```bash
npm run e2e:commission:dry -- --project-id test-e2e-01
```

**Behavior**:
- Validates CLI argument parsing
- Validates stage orchestration
- Skips all actual processing
- Generates report stub
- Exit code 0 on success

**Use Cases**:
- CI validation
- Local development wiring checks
- Pre-flight validation before full run

### Full Run (Interactive Mode)

```bash
npm run e2e:commission -- \
  --project-id 1 \
  --pdf path/to/rulebook.pdf \
  --bgg-url https://boardgamegeek.com/boardgame/13/catan
```

**Behavior**:
- Executes all stages with real API calls
- Prompts operator for gate confirmations
- Generates script candidate
- Confirms script as authoritative
- (Optional) Extracts images via HEPHAESTUS
- (Optional) Confirms component images
- Renders video (stub - not yet implemented)
- Verifies artifacts
- Generates `FIRST_FULL_E2E_RUN.md`

### Full Run (Non-Interactive Mode)

```bash
npm run e2e:commission -- \
  --project-id 1 \
  --pdf path/to/rulebook.pdf \
  --bgg-url https://boardgamegeek.com/boardgame/13/catan \
  --non-interactive \
  --confirm confirm_metadata \
  --confirm confirm_components \
  --confirm confirm_script
```

**Behavior**:
- Same as interactive mode
- Fails if any gate requires confirmation but not pre-provided
- Suitable for CI/automated runs

### Full Run with HEPHAESTUS

```bash
# Ensure HEPHAESTUS is enabled
export MOBIUS_ENABLE_HEPHAESTUS=true
export HEPHAESTUS_WORKSPACE=/path/to/hephaestus

npm run e2e:commission -- \
  --project-id 1 \
  --pdf path/to/rulebook.pdf \
  --use-hephaestus
```

**Behavior**:
- Includes image extraction stage
- Extracts images via HEPHAESTUS
- Prompts for component image confirmation
- Imports selected assets

## CLI Arguments

### Required
- `--project-id <id>` - Project ID in database

### Optional
- `--bgg-url <url>` - BoardGameGeek URL (if metadata not present)
- `--pdf <path>` - Path to rulebook PDF (required for non-dry-run)
- `--lang <code>` - Language code (default: `en`)
- `--use-hephaestus` - Enable HEPHAESTUS image extraction (default: false)
- `--dry-run` - Skip actual processing, validate wiring only
- `--non-interactive` - Fail if confirmations required but not pre-provided
- `--confirm <gateId>` - Pre-confirm a gate (can be repeated)
- `--confirm-file <path>` - JSON file with gate confirmations

### Confirmation File Format

```json
{
  "confirm_metadata": true,
  "confirm_components": true,
  "confirm_script": true,
  "confirm_component_images": true
}
```

## Stages

### 1. Ingestion
- **Purpose**: Ingest PDF and BGG metadata
- **Outputs**: Ingestion report with confidence scores
- **Gates Created**: `confirm_metadata`, `confirm_components`, etc.

### 2. Confirm Ingestion Gates
- **Purpose**: Operator confirms ingestion data quality
- **Inputs**: Ingestion report
- **Outputs**: Confirmed gate states
- **Blocking**: Yes - downstream stages require satisfied gates

### 3. Script Generation
- **Purpose**: Generate tutorial script candidate
- **Inputs**: Confirmed ingestion data
- **Outputs**: Script artifact with consistency validation
- **Blocking**: Yes - violations block confirmation

### 4. Confirm Script
- **Purpose**: Operator confirms script as authoritative
- **Inputs**: Script candidate
- **Outputs**: Authoritative script, `confirm_script` gate satisfied
- **Blocking**: Yes - TTS/storyboard/render require authoritative script

### 5. Image Extraction (Optional)
- **Purpose**: Extract component images via HEPHAESTUS
- **Inputs**: PDF path
- **Outputs**: ImageAsset DTOs (status=CLAIM)
- **Blocking**: No - only if `--use-hephaestus` flag set

### 6. Confirm Images (Optional)
- **Purpose**: Operator confirms extracted images
- **Inputs**: ImageAsset claims
- **Outputs**: Imported assets, `confirm_component_images` gate satisfied
- **Blocking**: Yes - if images extracted, must be confirmed before render

### 7. Render
- **Purpose**: Generate final MP4 + SRT
- **Inputs**: Authoritative script, confirmed images (if present)
- **Outputs**: MP4, SRT, render logs
- **Status**: STUB - not yet implemented

### 8. Verification
- **Purpose**: Verify all required artifacts exist
- **Checks**:
  - Gate confirmation records
  - Authoritative script
  - Imported images (if HEPHAESTUS used)
  - MP4 + SRT (when render implemented)

## Troubleshooting

### "Gate requires confirmation but --non-interactive is set"
**Cause**: Non-interactive mode requires all gates pre-confirmed  
**Solution**: Add `--confirm <gateId>` for each required gate or use `--confirm-file`

### "HEPHAESTUS not enabled"
**Cause**: `MOBIUS_ENABLE_HEPHAESTUS` not set to `true`  
**Solution**: Add to `.env`: `MOBIUS_ENABLE_HEPHAESTUS=true`

### "No ingestion report found"
**Cause**: Project has no ingestion data  
**Solution**: Run ingestion first or use a project with existing data

### "Script generation failed"
**Cause**: Gates not satisfied or missing required data  
**Solution**: Confirm all ingestion gates before script generation

### "API call failed: 500"
**Cause**: API server error  
**Solution**: Check API server logs, ensure database is accessible
