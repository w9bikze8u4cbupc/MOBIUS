## Problem

After the CI repair stack (PRs #385-#387) restored job instantiation and fixed governance checks, `build-and-qa` now runs but fails due to two pre-existing infrastructure issues:

1. **Ubuntu**: `jest: not found` — workflow installs deps only in `client/` but runs `npm test` at repo root where jest lives in root `devDependencies`
2. **macOS/Windows**: `Ensure FFmpeg` step fails — `FedericoCarboni/setup-ffmpeg@v2` has known issues with current runner images

## Solution

### 1. Add root dependency installation
Added `npm ci` at repo root before the test step so jest and other root devDependencies are available.

### 2. Upgrade FFmpeg setup action
Upgraded from `@v2` to `@v3` which downloads binaries directly from upstream sources instead of periodically-updated GitHub releases, fixing macOS and Windows installation failures.

## Changes
- `.github/workflows/ci.yml` — Add root `npm ci` step + upgrade FFmpeg action to v3
